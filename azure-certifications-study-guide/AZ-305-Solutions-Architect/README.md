## ğŸ“š Domain Details

### Domain 1: Identity, Governance, and Monitoring (25-30%)

| Topic | Key Concepts |
|-------|--------------|
| **Design identity solutions** | Entra ID, hybrid identity, B2B/B2C, conditional access |

| **Design monitoring** | Azure Monitor, Log Analytics, Application Insights |

### Domain 2: Data Storage Solutions (20-25%)

| Topic | Key Concepts |
|-------|--------------|
| **Design data storage** | Storage accounts, blob, files, disks, data lakes |
| **Design database solutions** | SQL, Cosmos DB, MySQL, PostgreSQL |
| **Design data integration** | Data Factory, Synapse, data movement strategies |
| **Design data protection** | Encryption, access control, data classification |

### Domain 3: Business Continuity (15-20%)

| Topic | Key Concepts |
|-------|--------------|

| **Design SLA requirements** | Composite SLAs, availability targets |

## ğŸ’¡ Exam Tips

1. **Understand design principles** - Not just how to do things, but when and why
2. **Think cost optimization** - Many questions involve cost-effective solutions
3. **Know trade-offs** - Understand when to choose one service over another
4. **Practice case studies** - The exam includes scenario-based questions
5. **Review reference architectures** - Azure Architecture Center is invaluable
