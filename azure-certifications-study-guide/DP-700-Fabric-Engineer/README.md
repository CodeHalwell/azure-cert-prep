## üìö Domain Details

### Domain 1: Implement and Manage Analytics Solution (25-30%)

| Topic | Key Concepts |
|-------|--------------|
| **Configure Fabric environment** | Workspaces, capacities, domains |
| **Create Lakehouse** | Delta tables, files, shortcuts |
| **Manage security** | Workspace roles, item permissions, RLS |
| **Design solution** | Medallion architecture, data mesh |

### Domain 2: Ingest and Transform Data (30-35%)

| Topic | Key Concepts |
|-------|--------------|
| **Data pipelines** | Copy activity, data flows |
| **Notebooks** | PySpark, transformations |
| **Dataflows Gen2** | Power Query, incremental refresh |
| **Real-time analytics** | Eventstream, KQL |

### Domain 3: Monitor and Optimize (20-25%)

| Topic | Key Concepts |
|-------|--------------|
| **Monitor performance** | Capacity metrics, workload management |
| **Optimize storage** | File formats, partitioning, V-Order |
| **Troubleshoot** | Spark jobs, pipeline runs |

### Domain 4: Semantic Models (15-20%)

| Topic | Key Concepts |
|-------|--------------|
| **Create models** | Tables, relationships, measures |
| **Design models** | Star schema, DAX |
| **Manage models** | Refresh, deployment |

## üí° Exam Tips

1. **Understand Lakehouse architecture** - Core to Fabric
2. **Know Delta Lake** - Transactions, time travel, optimization
3. **Practice with Notebooks** - PySpark transformations
4. **Understand medallion architecture** - Bronze/Silver/Gold
5. **Know when to use each tool** - Pipelines vs Notebooks vs Dataflows
